## 2. Deep Dive (核心概念與深度解析)
**The Ambiguity of the Threshold**
As discussed previously, the final neural network layer outputs probabilities. 
To convert $p=0.88$ into a binary $1$ (Edge) or $0$ (No Edge), a threshold $\tau$ must be chosen.
$$ \hat{Y}_{binary} = \begin{cases} 1 & p \geq \tau \\ 0 & p < \tau \end{cases} $$

If we deploy this model for Amazon Recommendations, choosing $\tau = 0.99$ means we only recommend products if we are absolutely certain (High Precision, Low Recall). We miss out on billions in potential revenue.
Choosing $\tau = 0.10$ means we spam the user with random irrelevant items (Low Precision, High Recall), destroying user trust.

**The ROC Curve (Receiver Operating Characteristic)**
Instead of picking one threshold and hoping for the best, the ROC Curve plots the True Positive Rate (Recall) against the False Positive Rate across *every single possible threshold* simultaneously.
The magical Area Under the Curve (ROC-AUC) score represents the probability that the model will rank a randomly chosen positive pair higher than a randomly chosen negative pair. It is strictly threshold-independent representation power.

$$ \text{FPR} = \frac{FP}{FP + TN} \quad ; \quad \text{TPR} = \frac{TP}{TP + FN} $$

## 3. Code & Engineering (程式碼實作與工程解密)
```python
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.metrics import roc_curve, auc, precision_recall_curve
import numpy as np

def plot_industrial_metrics(y_true: np.ndarray, y_probs: np.ndarray, save_path: str = "report.png"):
    """
    Kaggle-tier visualization plotting dual ROC and PR curves.
    """
    sns.set_theme(style="whitegrid")
    fig, axes = plt.subplots(1, 2, figsize=(16, 7))
    
    # ---------------- Plot 1: ROC Curve ----------------
    fpr, tpr, roc_thresholds = roc_curve(y_true, y_probs)
    roc_auc = auc(fpr, tpr)
    
    axes[0].plot(fpr, tpr, color='darkorange', lw=2.5, label=f'ROC curve (AUC = {roc_auc:.4f})')
    axes[0].plot([0, 1], [0, 1], color='navy', lw=2, linestyle='--')
    axes[0].set_xlim([0.0, 1.0])
    axes[0].set_ylim([0.0, 1.05])
    axes[0].set_xlabel('False Positive Rate', fontsize=14, fontweight='bold')
    axes[0].set_ylabel('True Positive Rate', fontsize=14, fontweight='bold')
    axes[0].set_title('Receiver Operating Characteristic', fontsize=16)
    axes[0].legend(loc="lower right", frameon=True, shadow=True)
    
    # ---------------- Plot 2: Precision-Recall Curve ----------------
    precision, recall, pr_thresholds = precision_recall_curve(y_true, y_probs)
    pr_auc = auc(recall, precision)
    
    # Calculate F1 scores across thresholds to find the OPTIMAL threshold
    # avoid division by zero
    f1_scores = 2 * (precision * recall) / (precision + recall + + 1e-8)
    optimal_idx = np.argmax(f1_scores)
    
    axes[1].plot(recall, precision, color='forestgreen', lw=2.5, label=f'PR curve (AUC = {pr_auc:.4f})')
    axes[1].scatter(recall[optimal_idx], precision[optimal_idx], marker='*', color='red', s=400, label=f'Max F1: {f1_scores[optimal_idx]:.3f}')
    axes[1].set_xlim([0.0, 1.0])
    axes[1].set_ylim([0.0, 1.05])
    axes[1].set_xlabel('Recall (Sensitivity)', fontsize=14, fontweight='bold')
    axes[1].set_ylabel('Precision (Positive Predictive Value)', fontsize=14, fontweight='bold')
    axes[1].set_title('Precision-Recall Architecture', fontsize=16)
    axes[1].legend(loc="lower left", frameon=True, shadow=True)
    
    plt.tight_layout()
    plt.savefig(save_path, dpi=300)
    print(f"Data Story successfully exported to {save_path}")

# Plotting the journey
print("Congratulations. You have completed the Ultimate GNN Textbook.")
```

*Engineering Note:* A high ROC-AUC on a violently imbalanced dataset (like 99% 0s, 1% 1s in link prediction) is notoriously misleading because the massive True Negative count artificially deflates the False Positive Rate. In production, always look at the Precision-Recall (PR) AUC curve as the absolute ground truth of performance.
